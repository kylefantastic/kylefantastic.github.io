So you want to create an object that takes a bunch of information and can manipulate that information in ways that you can determine on your own? Well I think you should be using a class. A class is a Ruby object that acts as a container for all the information you want it to hold. This can be categorical data, methods or variables that go into the class to compose its structure. To define a class you start with the def keyword (like a method) followed by the name of the class, the convention is to do this with the first letter capitalized. It will also have to have the keyword end at the end so Ruby knows where the class stops. After the method is defined there is typically a method to initialize the class, which is a special method called initialize. Within this method you can put any arguments, define any variables, or do any other work that you want the class to have that will be unique to the class.

If you want a variable to be accessible within the rest of the class, not just in the method you create it, you will need to make it an instance variable. All that this means is that when creating the variable, you put an @ sign before the variable name. Then it will be accessible anywhere in the class. You can also create methods within the class that can be called by appending them to the name of your new class instance with the dot syntax. (see below) These are known as instance methods because they are called when you call the instance of a class.
